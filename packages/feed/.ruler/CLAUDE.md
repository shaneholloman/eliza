# Feed AI Instructions (Single Source of Truth)

Edit rules in `.ruler/**` only. All other agent-facing files (ex: `AGENTS.md`, `CLAUDE.md`, MCP configs) are generated by Ruler and should not be edited manually.

## Non-negotiables

- **Keep apps thin**: apps are wiring-only (validate → call service → map errors).
- **Keep domain portable**: domain logic belongs in packages and must stay framework-agnostic (portable from Next → Elysia / `packages/core/*`).
- **Prefer local vendor docs**: use `docs/vendors/{vendor}` before guessing APIs.
- **Quality gate before “done”**:
  1. `bun run typecheck`
  2. `bun run lint` (must be zero warnings)
  3. `bun run build`
  4. Run relevant tests (see `packages/testing`)

## Code standards

- **Bun + TypeScript ESM**, 2-space indent, keep changes minimal and consistent.
- **Avoid broad `try/catch`**: don’t swallow errors; catch only expected errors and map them at boundaries (API/CLI). Otherwise fail fast.
- **No `any`**; avoid `unknown` (only as last resort, then narrow immediately).
- **Reuse before building**: search the codebase for existing patterns/utilities before adding new ones; prefer extending over duplicating.
- **Don’t invent behavior**: reuse existing code paths; don’t add fake “happy path” placeholders.
- **No synthetic data**: don’t generate fake data to “make it work”; build the real thing end-to-end (or fail with a clear TODO).
- **Documentation**: don’t create new docs/READMEs unless requested; do update existing docs when behavior/config changes (`README.md`, `.env.example`, etc.).
- **Env hygiene**: if you add/change an env var, update `.env.example` with required/optional + default notes; never bake secrets into code.
- **Secrets**: never commit or paste secrets (API keys, private keys, bearer tokens); don’t log sensitive values.
- **Scope discipline**: don’t fix unrelated lint/format or revert other WIP; keep changes scoped to the task.
- **Git**: default branch is `staging` (not `main`); commits are concise, imperative, prefixed (`feat:`, `fix:`, `chore:`).

## Tooling notes

- After pulling changes to `.ruler/**`, run `bun run ruler:apply`.
- If vendor docs are missing/outdated, run `bun run docs:generate`.
- Context7 is configured as a remote MCP server; if you use an API key for higher limits, set it locally (never in repo).
- For OpenAI Codex CLI project config/MCP: set `CODEX_HOME="$(pwd)/.codex"`.
