<Tip>
  **elizaOS** is six things in one repository: an operating system, a runtime,
  an app distribution layer, a hosted cloud, custom silicon, and an embodied
  robot stack. Every project on elizaOS lives in one (or more) of these.
</Tip>

## Three commands to a live agent

```bash
npx elizaos create my-agent
cd my-agent
bun install && bun run dev
```

That gets you the **Runtime** dimension. See [Quickstart](/quickstart) for full setup, or [Installation](/installation) for desktop / CLI install options.

## Why elizaOS?

<CardGroup cols={3}>
  <Card title="Plugin-first" icon="puzzle-piece">
    Connectors, model providers, and capabilities all ship as composable plugins under `@elizaos/*`.
  </Card>
  <Card title="Persistent memory" icon="brain">
    Agents remember conversations, learn from interactions, act on long-running goals.
  </Card>
  <Card title="Runs anywhere" icon="server">
    Local-only, desktop, mobile, Cloud, your own VPS, or directly on elizaOS hardware.
  </Card>
  <Card title="Open" icon="code-branch">
    Every package is open source. Extend, fork, contribute upstream.
  </Card>
  <Card title="Cloud optional" icon="cloud-arrow-up">
    Eliza Cloud handles auth, hosting, billing, and domains when you want it — never required.
  </Card>
  <Card title="Trains on your data" icon="microchip">
    Fine-tune Eliza-1 (2B–27B), evaluate against 70+ benchmarks, deploy on-device.
  </Card>
</CardGroup>
