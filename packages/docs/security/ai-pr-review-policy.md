# AI-Authored Pull-Request Review Policy

Applies to any PR whose code was substantially generated by an AI coding agent
(Claude Code, Codex, OpenCode, Cursor, Continue, GitHub Copilot Workspace, etc.).
SOC2 CC8.1 (change management) and CC9.2 (vendor / supply chain risk).

## Requirements

### 1. Human reviewer who has read the diff
Every AI-authored PR must be approved by at least one human reviewer who has
read the entire diff. A reviewer who merges without reading does not satisfy
this requirement and is treated as a process violation.

CODEOWNERS review counts toward this requirement only when the code-owner has
actually read the diff. Automated approval bots do not count.

### 2. `Generated-by` trailer
Each commit produced by an AI agent **should** include a trailer naming the
agent and (optionally) the model:

```
Generated-by: claude-code <opus-4-7>
Generated-by: codex <gpt-5>
Generated-by: cursor-composer
```

PRs may also be labeled `ai-authored` so the security team can audit the queue.

### 3. Per-package splits
A single PR must not span multiple top-level packages unless the change is
strictly mechanical (rename, dependency bump, version bump). When an agent
produces a sweeping refactor that touches many packages, split the PR per
package before requesting review. Reviewers are entitled to request a split.

### 4. No bypassing CI
AI-authored PRs follow the same required-checks list as human-authored PRs:
`typecheck`, `test`, `gitleaks`, plus any per-area checks. PRs that disable or
weaken CI to "fix" failures are rejected.

### 5. No silent secret rotation
Agents do not rotate, generate, or commit secrets. If an agent encounters an
exposed secret it must surface the finding to a human (preferably via the
security mailbox `security@elizaos.ai`). See `SECURITY.md`.

## Reviewer checklist

Reviewers reading an AI-authored diff should pay extra attention to:

- [ ] Spurious dependencies (new imports the project did not previously use).
- [ ] Patterns plagiarized from public training data — verify license
      compatibility for any block larger than a few lines that does not match
      the surrounding repo style.
- [ ] Defensive `try/catch` wrappers that swallow errors (AGENTS.md forbids).
- [ ] Fallback / "graceful degradation" that hides broken pipelines.
- [ ] Type widening (`any`, `unknown`, `as` casts) introduced to make a build
      pass.
- [ ] New ENV vars or feature flags not documented in `AGENTS.md` /
      `CLAUDE.md`.
- [ ] Unverified third-party URLs in install scripts or workflows.

## Audit trail

Every approval is captured in the PR's GitHub review history (already retained
by GitHub for the lifetime of the repo) and is sufficient for SOC2 evidence
of a human-in-the-loop review for AI-generated changes.
