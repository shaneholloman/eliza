The Training & Simulation track covers four overlapping subsystems:

- **Eliza-1** — open-weight Eliza model family (2B–27B), trained from Gemma 4 bases with native tool-calling SFT and APOLLO optimization.
- **Feed** — a satirical prediction-market simulation that generates social narratives and training data for Eliza agents.
- **Robot** — MuJoCo-based robotic embodiment with RL training and a real-hardware bridge.
- **Benchmarks** — 70+ evaluation harnesses spanning agent autonomy, tool-call correctness, voice, vision, and long-horizon reasoning.

## Fine-tuning collection runs

The fine-tuning plugin can collect Hugging Face training data, Feed-generated trajectories, natural app trajectories, scenario runs, test trajectories, eval artifacts, Eliza harness benchmarks, Cerebras comparisons, and model registry manifests into one collection directory.

Run a collection from the CLI:

```bash
bun run --cwd plugins/plugin-training train run-collection --tiers all
```

Each run writes `collection-manifest.json`, `README.md`, an analysis manifest, readiness report, and `analysis/index.html` for browsing data sources, trajectories, benchmarks, evals, model stats, and base-vs-trained comparisons. The collection root also gets `collection-index.json` and `collection-index.html` so saved runs can be rediscovered from the fine-tuning view or listed from the CLI:

```bash
bun run --cwd plugins/plugin-training train list-collections
```

To include app/runtime trajectories that were already exported, pass the sanitized and optional raw JSONL paths into the same collection run:

```bash
bun run --cwd plugins/plugin-training train run-collection \
  --natural-sanitized-jsonl ~/.local/state/eliza/training/natural/trajectories.sanitized.jsonl \
  --natural-raw-jsonl ~/.local/state/eliza/training/natural/trajectories.raw.jsonl \
  --natural-run-id app-run-1 \
  --natural-tasks response,action_planner
```

## Locked conventions

- **Training always uses APOLLO** optimizer. Memory-efficient; no alternatives ship.
- **Inference always applies every optimization** we have. DSPy artifacts auto-load on agent boot.
- **Recommended local model: `eliza-1`** — not enforced. Any HuggingFace model can be searched and downloaded from the models surface.
