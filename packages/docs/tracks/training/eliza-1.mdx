**Eliza-1** is the model family Eliza agents run locally by default. Built on Gemma 4 base models, fine-tuned for native tool-calling and agent loops, then put through a three-stage quantization pipeline for on-device inference.

## Shipping sizes

| Tier        | Model           | Base                  | Audience                          |
| ----------- | --------------- | --------------------- | --------------------------------- |
| Mobile / Laptop | `eliza-1-2b` | Gemma-4-E2B           | Phone / edge / laptop CPU + small GPU (smallest tier). |
| Workstation | `eliza-1-4b`    | Gemma-4-E4B           | Single consumer GPU.              |
| Workstation | `eliza-1-9b`    | Gemma-4-12B           | Mid-range workstation GPU.        |
| Cloud       | `eliza-1-27b`   | Gemma-4-31B           | Server-class GPU.                 |

All sizes are published under the [`elizaos/eliza-1`](https://huggingface.co/elizaos/eliza-1) HuggingFace repo with per-tier bundles. Legacy Qwen3 dense bases (0.6B / 1.7B / 4B) were dropped 2026-05-12 — they're incompatible with the mtp speculative-decode path.

## Optimizer

**APOLLO is locked.** Memory-efficient, no alternatives ship.

- 2B / 4B — `apollo_mini` (rank=1)
- 9B / 27B — `apollo`

## Quantization

Three real, shipping methods. The post-SFT pipeline runs all three:

| Method        | Target           | Technique                                                                 |
| ------------- | ---------------- | ------------------------------------------------------------------------- |
| **PolarQuant**| 4-bit weights    | Hadamard rotation ([arXiv:2603.29078](https://arxiv.org/abs/2603.29078)). fp16 norms in a sidecar safetensors file. |
| **TurboQuant**| 4-bit V-cache    | Online KV-cache quantization ([arXiv:2504.19874](https://arxiv.org/abs/2504.19874)). Pure-PyTorch reference plus a Triton-fused fast path. |
| **QJL**       | 1-bit K-cache    | JL-projected K-cache ([arXiv:2406.03482](https://arxiv.org/abs/2406.03482)). Vendored CUDA C++ extension. Gemma 4's MQA geometry uses dual head dims (512 global / 256 SWA), so the head_dim=128 QJL/Polar KV kernels don't apply — Gemma tiers run stock q8_0 KV. |

GGUF variants (`q3_k_m`, `q4_k_m`, `q5_k_m`, `q6_k`, `q8_0`) are also produced for llama.cpp consumption.

## Pipeline order

`base_bench → APOLLO SFT → fine_tuned_bench → PolarQuant + TurboQuant + QJL → quantized_bench`

Every stage emits a benchmark report so regressions land with evidence.

## Loading in an agent

Eliza-1 models load through the standard model-provider plugin surface (`@elizaos/plugin-llamacpp`, `@elizaos/plugin-ollama`, or direct FFI). Set the model id in your character's `settings.model`.
