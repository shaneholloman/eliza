**Eliza E1** is an open, pre-tapeout RISC-V AI phone SoC. It's the smallest end-to-end system used to prove project conventions, evidence gates, and tool setup before scaling to a production design. The repository ties together architecture contracts, RTL, cocotb/formal verification, QEMU/Renode software-facing smoke targets, FPGA evidence, packaging, and physical-design entry points.

E1 is **not** the final phone SoC. It's the lower-bound scaffold.

## When to pick this track

- You're contributing RTL, verification, or physical design.
- You're writing firmware or BSP code (OpenSBI, U-Boot, Linux, AOSP) that targets the chip.
- You're integrating a compiler or NPU runtime against the chip's contracts.
- You want to read the architecture before booting Eliza on real silicon.

## Status

Pre-tapeout. Expected blockers (missing EDA tools, simulators, Android BSP, hardware) are recorded as explicit evidence artifacts. The repo treats them as gates, not hidden failures.
