Channels are plugins. To bring an agent into a platform, install the relevant `@elizaos/plugin-*` package, add it to your character's `plugins` array, and set the required environment variables.

## Pattern

Every connector follows the same shape:

```ts
// character.ts
export const character: Character = {
  name: 'Eliza',
  plugins: [
    '@elizaos/plugin-sql',
    '@elizaos/plugin-anthropic',
    '@elizaos/plugin-discord',  // <- add the channel plugin
  ],
  settings: {
    // optional per-plugin overrides
  },
};
```

```bash
# .env — credentials per channel
DISCORD_API_TOKEN=...
DISCORD_APPLICATION_ID=...
```

Restart the agent. The connector registers a service, opens its websocket / webhook / poll loop, and starts forwarding messages into the runtime.

## Owner vs. agent

Most connectors distinguish the **owner** (the human who configured the agent) from the **agent** itself. Owner DMs can trigger privileged actions; non-owner traffic is treated as public. See per-connector docs for owner-detection details.

## Multiple channels at once

An agent can listen on as many channels as you load. Each connector contributes to the same memory store, so context follows the user across channels.
