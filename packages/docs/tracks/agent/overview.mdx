An **Eliza agent** is an `AgentRuntime` instance wrapped around a **Character** — a configuration file describing personality, knowledge, capabilities, and the connectors it speaks through. The runtime handles message ingestion, action selection, memory, evaluator passes, and provider context injection. Everything else is a plugin.

## What you build

A single agent, addressable from one or more channels (Discord, Telegram, Slack, X, WhatsApp, iMessage, the elizaOS app — see [Connectors](/tracks/agent/connect-channels)). It can hold long-running goals, remember conversations, run tools, and act on a schedule.

## Three commands

```bash
npx elizaos create my-agent
cd my-agent
bun install && bun run dev
```

The scaffolded project includes a `character.ts`, a default plugin stack, and a dev entry point. See the [CLI](/cli/overview) reference for what each command does.
